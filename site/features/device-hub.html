About this feature

Why it matters ›

Create puts keys in one tab; strangers need the story, returners need a control center. The hub names the two storage layers and surfaces the next action (save, notice, revoke, vouch).

Design decisions ›

Status line — segmented counts (network, saved, pinned, notice); tap expands On this device. Saved cards — Use keys, Open scan, relabel/remove. Pinned scans — public bookmarks only (hc_device_pins). Search — inline in hub; filters local rows only (not resolver-wide). Backup import — decrypt .hcbackup.json into hc_wallet. Focus mode — hide intro sections; keep hub + documentation.

Safety · privacy · security ›

Private keys stay in browser storage; pins never hold signing material. Search is client-side over data you already saved. Labels are yours — rows show @handle + profile id so synced browser data cannot lie silently.

Limits ›

Not cloud sync or multi-device accounts. Browser profile sync (Safari/Chrome) may copy localStorage incompletely — prefer explicit save per machine. No directory search for other people’s cards.

Future directions ›

Optional activity log on device, clearer multi-tab key handoff, federated read — still no operator custody of owner keys.
