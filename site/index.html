            Studio

            One use · status plate

            Our first deployed object is a studio door plate: a QR that answers with live hours, open/closed, and “done for the night,” updatable without reprinting. The same primitive works on shops, booths, and rehearsal rooms.

            • Door scan “Open Thu–Sun · back at 7” before someone pulls the handle
            • Run state Flip to closed or done for the night from /created/
            • Special hours One signed line for open studio, critique night, or a holiday
            • Guest wayfinding Object name and status when the studio shares a building entrance
            • After revoke Plate still resolves; scanners see revoked, not a dead 404

            Design choices

            Engineering decisions - each tag names a concrete mechanism, not marketing copy.

            • Optional Scanning needs no account. Create a live object only when you want one - keys stay in your browser, not on our servers.
            • Revocable Owner signs revoke with an Ed25519 key held locally; the resolver verifies the signature and updates server-side status for that qr_id. The printed URL stays - the live answer changes.
            • No scan tracking The resolver returns object state, not a trail of who scanned, when, or where. No scan analytics by default.
            • Not legal ID Signed public status for communities - not government ID, KYC, employment proof, or a hidden trust score.
