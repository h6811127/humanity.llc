Authority stays clear

Two signed streams on one physical node — game state and safety state — from different keys, shown together but never confused.

The game can reward a player for finding an AED cabinet. But only the building maintainer can sign that the device is inspected and in service. A single QR sticker pulls from both streams; the scan page labels which authority is speaking.

What changes over time

Player reports issue

  • Player scans, sees "pad cover cracked"
  • Taps report → relay reaches building ops
  • Game awards lore for noticing the problem
  • No game points for declaring equipment safe

Maintainer pauses node

  • Maintenance signs: out of service
  • Game map marks node paused
  • Safety copy outranks game copy
  • Players cannot capture a paused node

Fix verified → route unlocks

  • Maintainer signs: back in service
  • Game unlocks the next trail segment
  • Local map shows the new route
  • Repair history stays in maintenance log

Two streams, one sticker

  • Game organizer key Signs faction state, lore, checkpoints, player discovery counts — never signs safety status
  • Maintainer key Signs inspection dates, in/out of service, repair history — never signs game rewards
  • One QR sticker Resolves to a scan page that shows both streams, labeled by signer — revocable independently

The boundary

Game can

  • Reward finding the node
  • Award lore for noticing problems
  • Unlock routes after verified repairs
  • Track faction control of areas

Game cannot

  • Declare equipment safe
  • Override a maintenance pause
  • Create urgency to drive engagement
  • Sign anything with the maintainer key

Play helps people notice infrastructure worth caring for — but the maintainer’s signature is the only safety truth on the sticker.
