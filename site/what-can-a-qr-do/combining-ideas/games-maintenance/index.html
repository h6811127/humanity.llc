One scan can host evolving objects, rotating messages, and local secrets while keeping maintainer-signed safety authority separate from public play.

What the scanner sees now

  • Node status: in service
  • Last maintainer check: today, 09:14
  • Local object: River Lantern Shell
  • Clue window: visible for the next 48 hours

Public-play layer

  • Evolving objects change after enough voluntary contributions
  • Rotating messages and ephemeral drops create timed neighborhood moments
  • Collaborative discovery unlocks the next object without points, XP, or leaderboards

Safety boundary

  • Maintainer-signed status stays separate from lore and clue state
  • If signatures conflict, safety copy overrides the object copy
  • Resolver stores unlock state, not a movement history

Signed streams behind this card

  • Maintainer stream In service / out of service, last verified, repair notes
  • Object stream Current object form, rotating message, and local unlock conditions
  • Trust chain Which nearby object vouched, whether the clue is revoked, and what appears next

Example object chain

Temporary drop

  • A QR appears near the riverwalk for 48 hours
  • It resolves to a hidden object instead of a generic reward screen
  • Stewards can revoke it if the physical site needs to change

Collaborative unlock

  • If enough people scan the first object, a second object wakes up in Czech Village
  • The resolver only stores whether the threshold was met
  • No account or player dossier is required to participate

Trust gate

  • A third object appears only after another local object vouches for it
  • Revocable physical clues keep the chain grounded in real space
  • The object can disappear cleanly without preserving scan histories

The strongest mechanic is not points; it is anonymous participation in shared urban memory while the maintainer still signs safety-critical truth.
